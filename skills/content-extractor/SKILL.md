# 内容编辑与知识提炼技能

## 触发条件
当用户要求从长字幕中提取精华片段、高光片段、核心观点、内容提炼时使用此技能。

## 输入格式

用户输入为长字幕文本（SRT 或去时间轴的纯文本）
内容通常为：知识讲座 / 技术分享 / 访谈 / 课程等
文本可能非常长（数千行）

## 核心任务流程（必须按顺序执行）

### Step 1：主题识别（必须优先完成）

- 通读全文，识别内容中的**核心主题（Topics）**
- 主题划分标准：
  - 每个主题应具有**独立观点或讨论方向**
  - 同一主题下的内容在语义上高度相关
- 输出合理的主题数量（通常 3～10 个，不要过多或过少）
- 每个主题用一句话命名（简洁清晰）

### Step 2：高光片段提取（核心任务）

针对每一个主题：

- 从原文中提取**精彩片段（Highlights）**
- 片段要求：
  - 可以**不连续摘取**
  - 可以跨段组合，但必须保持语义完整
  - 保持**原文表达**（允许轻微删减冗余，但不可改写原意）
- 每个主题提取 4~10 个高光片段

## 高光片段筛选标准（非常重要）

### 优先选择以下内容：

- 有**洞察力/观点表达**（不是铺垫或废话）
- 有**"金句感"或传播力**
- 有**信息密度高的解释或总结**
- 有**具体案例 / 类比 / 反直觉观点**
- 对观众"有收获"的内容

### 避免选择以下内容：

- 过渡语（比如"我们接下来讲…"）
- 重复内容
- 信息密度低的描述

## 输出结构（严格遵守）

```
主题1：xxx
- 片段1：xxx
- 片段2：xxx
- 片段3：xxx

主题2：xxx
- 片段1：xxx
- 片段2：xxx
...
```

## 语言要求

- 保持原文语言（不要翻译）
- 不要改写风格
- 仅在必要时删除冗余词，使片段更紧凑

## 质量要求

- 主题划分要**覆盖全文核心内容**
- 每个主题之间要**区分明显，不重叠**
- 高光片段必须"值得单独传播"
- 避免片段之间内容重复

## 禁止行为

- 不要输出总结性段落
- 不要解释你的思考过程
- 不要添加原文中不存在的信息
- 不要输出与任务无关内容

## 输出要求

- 只输出最终结构化结果
- 不要添加说明文字
