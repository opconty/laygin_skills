# 周报生成技能

## 触发条件
当用户要求生成周报、工作总结、本周汇报时使用此技能。

## 工作流程

### 1. 收集本周工作信息
用户会直接发送本周工作内容给你，你基于用户提供的内容生成周报。

**用户可能发送的内容格式：**
- 按日期列出的工作项
- 按项目分类的任务列表
- 简短的工作描述
- 口语化的工作汇报

无论用户以什么格式提供信息，都需提取并整理到周报结构中。

### 2. 生成周报内容
周报应包含以下结构：

**格式：**
```markdown
# 周报 - [日期范围]

## 一、本周工作总结
### 1. 已完成工作
- 项目/任务名称
  - 工作内容
  - 完成情况/成果

### 2. 进行中工作
- 当前进度
- 预计完成时间

### 3. 问题与风险
- 遇到的问题
- 解决方案或需要的支持

## 二、下周工作计划
- 计划完成的任务
- 重点事项

## 三、其他事项
- 团队协作
- 学习成长
- 建议与反馈
```

### 3. 根据用户角色调整语气
- 如果是开发：侧重技术实现、代码提交、bug 修复
- 如果是产品：侧重需求分析、原型设计、评审会议
- 如果是设计：侧重设计稿、用户研究、设计规范

### 4. 输出
将生成的周报保存到 `reports/weekly-report-[日期].md`

## 注意事项
- 保持简洁，避免空话套话
- 尽量用数据和事实说话
- 突出成果和价值
- 如果信息不足，先向用户提问补充
